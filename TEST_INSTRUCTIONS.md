# 测试指令集

启动交互式控制台后逐条粘贴测试:

```bash
python decompose.py                 # 默认 deepseek
python decompose.py --provider glm  # 换提供商
```

## 一、正常指令 — 简单搬运

| 指令 | 预期拆解 |
|---|---|
| 把水壶放到桌子上 | Pick → Carry → Place(logic1, 表面) |
| 把遥控器放到沙发上 | Pick → Carry → Place(logic1, 表面) |
| 把杯子放在盘子的左边 | Pick → Carry → Place(logic0, coef=to the left of) |
| 把椅子搬到窗户前面 | Pick → Carry → Place(logic0, coef=in front of) |

## 二、正常指令 — 含隐含步骤(开门/关门)

| 指令 | 预期拆解 |
|---|---|
| 把牛奶放进冰箱 | Pull(开冰箱门) → Pick → Carry → Place(logic2, 空间) → Push(关冰箱门) |
| 把袜子放进抽屉 | Pull(开抽屉) → Pick → Carry → Place(logic2) → Push(关抽屉) |
| 从冰箱里拿出可乐放到餐桌上 | Pull(开门) → Pick → Carry → Place(logic1) → Push(关门) |

## 三、正常指令 — 覆盖其他原子操作

| 指令 | 预期涉及的操作 |
|---|---|
| 把外套挂在衣架上 | Pick → Carry → Hang (A_006) |
| 把茶壶里的水倒进杯子 | Pick → Carry → Pour (A_007) |
| 按下电梯按钮 | PressButton (A_008) |
| 用抹布擦桌子 | Pick → Carry → Wipe (A_009) |
| 把充电器插到插座上 | Pick → Carry → Insert (A_010) |
| 把门旋转到打开状态 | Rotate (A_011) 或 Open |
| 用勺子从锅里舀一勺汤 | Pick → Scoop (A_012) |
| 用筷子搅拌碗里的鸡蛋 | Pick → Stir (A_013) |
| 把平底锅里的煎饼翻个面 | Turn (A_014) |
| 用门禁卡刷一下读卡器 | Pick → Swipe (A_015) |
| 关掉电视 | Close (A_016) 或 PressButton |
| 打开笔记本电脑 | Open (A_017) |

## 四、指令不明确 — 预期返回 `[指令不明确] + 原因`

| 指令 | 不明确之处 |
|---|---|
| 把水壶放到那边 | 「那边」不是具体的表面/空间/方位 |
| 把它拿起来 | 「它」指代不明,不知道拿什么 |
| 帮我把东西收拾一下 | 「东西」「收拾」都没有具体对象和目标 |
| 把书放过去 | 「过去」未指明放到哪里 |
| 放到桌子上 | 没有说放什么物体 |
| 把杯子放在盘子旁边 | 「旁边」需明确是左/右/前/后哪个方位(方位 coef 只有四个可选值) |

## 五、无法完成 — 预期返回 `[无法完成] + 原因`

| 指令 | 超出能力的动作 |
|---|---|
| 把苹果切成两半 | 没有切割操作 |
| 把纸撕碎扔进垃圾桶 | 没有撕的操作(注意:观察它是否把后半段「扔进垃圾桶」也正确判定为不可独立完成) |
| 把衣服叠好放进衣柜 | 没有折叠操作 |
| 把鸡蛋打进碗里 | 没有敲破/打蛋的操作 |
| 给我讲个笑话 | 不是物理操作任务 |
| 把房间的灯泡换掉 | 没有拧卸/更换类操作(Rotate 只针对铰链) |

## 六、边界用例 — 观察模型如何取舍

这些指令介于类别之间,答案不唯一,重点看返回是否合理、原因是否讲得通:

| 指令 | 观察点 |
|---|---|
| 烧一壶水 | 可能拆成 接水(Pour?)→放到底座→PressButton,也可能判定信息不足或超能力范围 |
| 把咖啡搅拌均匀 | 没说用什么搅拌(spoon 需推断)——看它是合理推断勺子还是判定不明确 |
| 打开窗户再把花盆放到窗台上 | 复合指令:Open + Pick/Carry/Place 两段动作是否都拆出且顺序正确 |
| 把桌上的三本书都放到书架上 | 多个同类物体:看是否拆成三轮 Pick/Carry/Place |
| 把冰箱门打开 | 单步指令:应只有 Pull 或 Open 一步,不应多拆 |
